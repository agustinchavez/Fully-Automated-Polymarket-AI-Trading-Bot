Polymarket Research & Trading Bot

Version 1.0.0 · MIT License · Python 3.9+

What is this?

A production-grade, AI-powered autonomous trading agent for Polymarket prediction markets. It discovers markets, gathers authoritative evidence from 26 research connectors, generates calibrated probability forecasts using a multi-model LLM ensemble, enforces strict risk controls, and executes trades with fractional Kelly sizing. Includes a 10-phase feature set: backtesting, domain specialists, cross-platform arbitrage, whale intelligence, continuous learning, and graduated live deployment.

Key Capabilities

SRC

Market Discovery

Auto-scans Polymarket's Gamma API for markets matching volume, liquidity, spread, and category filters.

RES

26 Research Connectors

DuckDuckGo (default, free), FRED, CoinGecko, Congress.gov, CourtListener, Open-Meteo, Kalshi, Metaculus, and 18 more structured data sources.

AI

Multi-Model Ensemble

Queries GPT-4o, Claude Sonnet 4.6, and Gemini 2.0 Flash in parallel with adaptive weighting and Platt calibration.

RSK

15-Check Risk Engine

Kill switch, drawdown management, portfolio limits, min edge, evidence quality gates, VaR gate, and more. ALL must pass.

KEL

Kelly Position Sizing

Fractional Kelly criterion with confidence scaling, drawdown heat multiplier, uncertainty adjustment, and liquidity-aware caps.

WHL

Whale Intelligence

5-dimension quality scoring (S/A/B/C tiers), conviction signals, Smart Money Index, and quality-weighted edge adjustment.

BKT

Historical Backtesting

Full pipeline replay with LLM cache, Brier/Sharpe/calibration metrics, and A/B comparator with paired t-tests.

ARB

Cross-Platform Arbitrage

Polymarket vs Kalshi spread detection, complementary YES+NO arb, and correlated mispricing scanner.

Trading Pipeline

Every trading cycle follows this deterministic pipeline. Each step must succeed for the next to proceed:

1

Market Discovery

Gamma API scan → filter by volume, liquidity, spread, category → classify market type

2

Research & Evidence Gathering

Build site-restricted queries → fetch from whitelisted sources → LLM extracts structured evidence with citations

3

Feature Engineering

30+ features from market data, orderbook, trade history, and evidence quality scores

4

LLM Ensemble Forecast

GPT-4o + Claude Sonnet 4.6 + Gemini 2.0 Flash queried in parallel → trimmed mean / adaptive weighted aggregation

5

Calibration

Platt scaling shrinks extremes toward 0.5 → low-evidence penalty → contradiction penalty → historical calibration learning

6

Edge Calculation

Edge = model_prob − implied_prob → deduct transaction costs → determine BUY_YES or BUY_NO direction

7

Risk Check (15 rules)

Kill switch, drawdown, max stake, daily loss, open positions, min edge, liquidity, spread, evidence quality, confidence, implied prob floor, edge direction, market type, portfolio exposure, timeline

8

Position Sizing

Fractional Kelly × confidence × drawdown heat × timeline × volatility × regime × category multipliers

9

Order Execution

Build limit/market/TWAP/iceberg order → route through triple dry-run gate → submit to CLOB or simulate

Pre-Research Filter

Before spending API credits on research, markets are scored (0-100) on volume, liquidity, recency, and category. Markets scoring below filter_min_score (default 25) are skipped. Markets of type UNKNOWN are blocked entirely.

Research Engine

The research module implements a source-first, whitelisted search pipeline to ensure evidence comes from authoritative sources.

Query Strategy (4 phases)

1
Site-Restricted Queries — Searches primary authoritative domains per market type (e.g., site:bls.gov CPI January 2026)
2
Metric-Specific Queries — Date-scoped searches for specific data releases and official numbers
3
Confirmation Queries — Secondary outlets (Reuters, Bloomberg) for corroboration
4
Contrarian Queries — Deliberately searches for opposing evidence to reduce overconfidence bias

Evidence Extraction

An LLM processes fetched sources to extract structured evidence. Every bullet must include:

{
  "text": "CPI-U increased 3.1% YoY in January 2026",
  "metric_name": "CPI-U YoY",
  "metric_value": "3.1",
  "metric_unit": "percent",
  "metric_date": "2026-01-31",
  "confidence": 0.97,
  "citation": {
    "url": "https://www.bls.gov/...",
    "publisher": "Bureau of Labor Statistics",
    "authority_score": 1.0
  }
}

Evidence Quality Scoring

Quality is computed independently of LLM self-assessment using:

  • Source recency — Penalty for stale data (>7 days old, severe >30 days)
  • Domain authority — Government sources score 1.0, news outlets 0.6-0.8
  • Cross-source agreement — Multiple sources confirming the same metric
  • Numeric density — Bonus for evidence with concrete numbers, dates, and units
  • Contradiction detection — Conflicting sources reduce confidence

Signal Aggregation

The SignalStack aggregates signals from all sources into a unified context for the LLM prompt: consensus prices (Kalshi, Metaculus), behavioral signals, economic indicators, and cross-platform spreads. The compute_signal_confluence() function adjusts Kelly sizing based on signal agreement.

Blocked Domains

The following domains are always excluded: wikipedia.org, reddit.com, medium.com, twitter.com/x.com, quora.com, and other user-generated content sites.

Prompt Versions

  • v1 (default) — Standard: question + evidence + market features → probability + confidence + reasoning
  • v2 (opt-in) — Structured reasoning chain: base rate → evidence for/against → adjustment → final probability

Question Decomposition

When enabled (decomposition_enabled: true), complex questions are split into up to 3 sub-questions using gpt-4o-mini. Each sub-question is forecast independently, then recombined using weighted combination.

Domain Specialists

SpecialistData SourceModeDescription
WeatherOpen-Meteo GFSBypass31-member GFS ensemble → fraction exceeding threshold (bypasses LLM entirely)

Aggregation Methods

  • Trimmed Mean (default) — Removes the highest and lowest forecasts, averages the rest. Robust to outlier models.
  • Median — Takes the middle forecast. Completely resistant to outliers.
  • Weighted — Uses per-model weights (static or adaptive). Best when model quality is known.

Graceful Degradation

If some models fail (API errors, timeouts), the ensemble degrades gracefully. It requires min_models_required (default 1) to succeed. If all models fail, it falls back to a single-model query to fallback_model (default: GPT-4o).

Prompt Structure

Each model receives: the market question, market type, evidence summary, top evidence bullets, contradictions, and market features (volume, spread, implied probability, days to resolution). The model must return a probability (0-1), confidence level (LOW/MEDIUM/HIGH), reasoning, and invalidation triggers.

Adaptive Model Weights

The AdaptiveModelWeighter module tracks per-model, per-category Brier scores from resolved markets. It computes inverse-Brier weights so that models with better track records in a specific category receive more influence. Uses Bayesian smoothing to blend learned weights with priors, requiring 50+ samples for full confidence.

Calibration

Raw LLM probabilities are systematically adjusted for better real-world calibration.

Platt Scaling (Default)

Applies logistic compression with shrinkage toward 0.5. This counteracts the tendency of LLMs to produce overconfident extreme probabilities.

Formula:

calibrated = sigmoid(a × logit(raw) + b)

Where a < 1 provides shrinkage (pulling extremes inward) and b provides a base-rate shift.

Adjustments Applied

  • Shrinkage — Extreme probabilities (>0.95 or <0.05) are pulled toward 0.5
  • Low-Evidence Penalty — Up to 15% penalty when evidence quality is below threshold
  • Contradiction Penalty — Each contradiction reduces confidence proportionally
  • Ensemble Disagreement — Wide spread among models triggers additional shrinkage

Historical Calibration Learning

The CalibrationFeedbackLoop records every (forecast, outcome) pair when markets resolve. After 30+ samples, the bot trains a logistic regression to learn optimal calibration parameters from its own track record, becoming more accurate over time.

Edge Calculation

Edge measures the difference between the bot's model probability and the market-implied probability.

Core Formula:

raw_edge = model_probability − implied_probability

net_edge = |raw_edge| − transaction_cost_pct

Direction Logic

  • If model_prob > implied_prob → BUY YES (the market underestimates probability)
  • If model_prob < implied_prob → BUY NO (the market overestimates probability)

Transaction Costs

Costs include entry fees (transaction_fee_pct, default 2%) and gas costs. The bot uses a hold-to-resolution model (single-leg entry, no exit trade), so only entry costs are deducted.

Expected Value

EV per dollar = (model_prob × $1 − cost) / cost. Only positive net-EV trades pass risk checks.

Risk Management

The risk engine enforces 15 independent checks. A trade is only allowed if ALL pass. Any single violation blocks the trade.

1
Kill Switch
Manual global halt; immediately stops all trading
2
Drawdown Kill
Auto-kill when drawdown exceeds max (default 20%)

Position Sizing (Kelly Criterion)

Position sizes are computed using the fractional Kelly criterion with multiple adjustment layers.

Kelly Formula (Binary Outcome):

f* = (p × b − q) / b

Where: p = model probability, q = 1−p, b = odds = (1/cost) − 1

Adjustment Multipliers

MultiplierSourceRangePurpose
Kelly FractionConfig0.25 (quarter-Kelly)Reduces full Kelly for real-world variance
ConfidenceLLM ForecastLOW=0.5, MED=0.75, HIGH=1.0Scale by forecast confidence
Drawdown HeatDrawdownManager0.0 – 1.0Reduce sizing during drawdowns
TimelineTimelineAssessment0.5 – 1.3Adjust for resolution timing
VolatilityMarket microstructure0.0 – 1.0Reduce in volatile markets
RegimeRegimeDetector0.5 – 1.0Reduce in HIGH_VOLATILITY regime
CategoryConfig per-categoryMACRO=1.0, CORP=0.75, ELEC=0.5Category-specific risk appetite

Caps

  • max_stake_per_market — Hard cap per trade (default $50)
  • max_bankroll_fraction — No trade exceeds 5% of bankroll
  • Liquidity cap — Stake capped at 5% of available market liquidity

Order Execution

The execution layer builds orders and routes them through safety checks before submission.

Order Types

  • Limit Orders — Default; placed at calculated price with TTL (default 300s)
  • Market Orders — Immediate fill at current price; used for urgent entries
  • TWAP — Time-Weighted Average Price; splits large orders into 5 slices over 30s intervals
  • Iceberg — Hides order size; shows only 20% of total, refills as filled. Triggered at $500+ orders

Triple Dry-Run Gate

Three independent gates prevent accidental live trading:
  1. order.dry_run flag on the order object itself
  2. config.execution.dry_run in config.yaml (default: true)
  3. ENABLE_LIVE_TRADING environment variable (must be "true")

ALL THREE must allow live trading for any order to be submitted to the CLOB.

Whale / Smart-Money Scanner

The Whale Tracker is a comprehensive smart-money intelligence system that monitors the top 15 Polymarket leaderboard wallets and generates actionable signals. It provides real-time visibility into what the most profitable traders are doing.

Architecture

The scanner runs on a 15-minute cycle within the bot's main engine loop:

  • Phase 1 – Fetch: Queries positions for all 15 tracked wallets via the Data API (up to 200 positions each)
  • Phase 2 – Score: Computes a composite quality score (0-100) per wallet based on P&L, win rate, and activity
  • Phase 3 – Delta Detection: Compares current positions against the previous snapshot to identify new entries, exits, and size changes (>10% threshold)
  • Phase 4 – Conviction Signals: Groups positions by market+outcome and generates conviction signals where 2+ whales agree

Smart Money Index (SMI)

A proprietary aggregate indicator (0-100) computed from all active whale signals, weighted by position size:

SMI RangeInterpretationDescription
70-100Very BullishStrong net positive whale conviction across markets
60-70BullishModerate positive bias from smart money
40-60NeutralMixed signals — no clear directional consensus
30-40BearishModerate negative bias from smart money
0-30Very BearishStrong net negative whale conviction
How SMI is Calculated:

SMI = (Σ bullish_conviction × position_usd) / (Σ all_conviction × position_usd) × 100. A value of 50 means equal bullish and bearish conviction-weighted capital.

Profit FactorUp to 15 ptsBonus when whales are in profit (entry < current price)

High Consensus Alerts

Net Whale Flow

Tracks aggregate capital entering vs. exiting whale positions:

  • Flow In: Sum of $ from NEW_ENTRY and SIZE_INCREASE deltas
  • Flow Out: Sum of $ from EXIT and SIZE_DECREASE deltas
  • Net Flow: Flow In - Flow Out (positive = net accumulation, negative = net distribution)

API Connectors

Gamma API (Market Discovery)

The GammaClient connects to Polymarket's public market-listing API at https://gamma-api.polymarket.com. Used for: listing markets, fetching metadata (question, category, end date, resolution source), and getting pricing snapshots.

The CLOBClient connects to https://clob.polymarket.com for real-time orderbook data (bids/asks), trade history, spread calculations, and order placement via the py-clob-client signing library.

Kalshi API (Cross-Platform Arbitrage)

The KalshiClient connects to Kalshi's API with RSA-PSS authentication for cross-platform price comparison. Supports paper mode by default. The MarketMatcher maps Kalshi markets to Polymarket equivalents using manual mappings + keyword/Jaccard similarity matching.

WebSocket Feed

The WebSocketFeed subscribes to real-time price ticks for monitored markets, providing live data for the dashboard and microstructure analysis.

Rate Limiter

Token-bucket rate limiters per provider prevent API abuse with configurable per-endpoint limits. Each LLM provider, search engine, and research connector has its own rate limiter bucket.

Circuit Breakers

Each API provider has its own circuit breaker (CLOSED → OPEN → HALF_OPEN state machine). After repeated failures, the circuit opens to prevent cascading failures, then allows one test request after a recovery period. Per-model circuit breakers are wired into the ensemble's _query_* methods.

Historical Backtesting

Full pipeline replay engine for validating strategy changes against resolved markets.

Architecture

  • Separate SQLite DB — data/backtest.db (SCHEMA_VERSION=2), completely isolated from live trading
  • LLM Response Cache — SHA-256 keyed by (question + model + template_version), excludes volatile features. Cache-hit avoids re-running LLM calls
  • Mock Evidence — No web search during backtests; forecaster relies on parametric knowledge only
  • Execution Simulation — contracts = stake / entry_price; pnl = (exit_price - entry) * contracts

A/B Comparator

Compare two backtest runs side-by-side with paired t-test on overlapping market P&L. Significance levels: none, weak (p<0.10), moderate (p<0.05), strong (p<0.01). Config differences are highlighted automatically.

The Backtest tab shows run history, equity curves, calibration charts, trade-level detail, and the A/B comparator. Cache stats show hit rate to monitor LLM cost savings.

Continuous Learning & Self-Improvement

A suite of self-improving features that analyze past performance and automatically adjust strategy. All disabled by default.

Post-Mortem Analysis

After markets resolve, the PostMortemAnalyzer generates per-trade analysis (what went right/wrong, evidence quality, model accuracy) and weekly summaries with lessons learned.

Evidence Source Quality

The EvidenceQualityTracker records outcomes per domain and computes effective weights (0.5–1.5x). Sources that consistently provide accurate evidence get boosted; unreliable sources get penalized.

Parameter Optimizer

Tunes 6 parameters (min_edge, kelly_fraction, max_stake, etc.) using random perturbations (±20%) and paired t-tests. Results are human-approval-only — the optimizer suggests changes but never auto-applies them.

Smart Calibration Retraining

Automatically re-fits the calibration curve when triggered by: resolution count threshold, Brier score degradation, or specialist model changes. Uses 80/20 A/B holdout validation (SHA-256 split) to ensure the new calibration outperforms the old.

Cross-Platform (Polymarket vs Kalshi)

The CrossPlatformArbScanner computes 4-directional spreads between matched Polymarket and Kalshi markets. When spread exceeds min_arb_edge (default 3%), paired trades are executed on both platforms simultaneously.

Glossary

Forecasting & Accuracy

TermDefinition
Brier ScoreForecast accuracy metric: mean((forecast - outcome)^2). Ranges 0 (perfect) to 1 (worst). Superforecaster level: 0.10. Random guessing: 0.25. The bot requires Brier < 0.25 to unlock live trading.
CalibrationHow well predicted probabilities match actual outcomes. A calibrated forecaster predicting 70% should be right ~70% of the time. The dashboard plots calibration curves (predicted vs actual).
Base RateHistorical frequency of an event type (e.g., "incumbents win re-election ~70%"). The bot maintains 56 base rate patterns across 7 categories.
EnsembleRunning multiple AI models (GPT-4o, Claude Sonnet 4.6, Gemini 2.0 Flash) in parallel and combining forecasts. Aggregation: trimmed mean, median, or weighted average.
Platt ScalingCalibration technique using logistic regression: calibrated = sigmoid(a × logit(raw) + b). Shrinks extreme probabilities toward 0.5.
Implied ProbabilityProbability derived from market price. A YES token at $0.65 implies 65% probability.

Edge & Trading

TermDefinition
EdgeDifference between model probability and market price, minus fees. Formula: net_edge = |model_prob - market_price| - fees. Minimum 4% required by default.
Kelly CriterionOptimal bet sizing formula: f* = edge / odds. The bot uses fractional Kelly (default 25%) with 7 adjustment multipliers (confidence, drawdown, timeline, volatility, regime, category, liquidity).
SlippageDifference between expected and actual fill price, measured in basis points (bps). 100 bps = 1%. Tracked by execution strategy (Simple, TWAP, Iceberg).
TWAPTime-Weighted Average Price; splits large orders into 5 time-weighted child orders to minimize market impact.
Iceberg OrderHides full order size; shows only 20% to the market, refills as filled. Used for very large orders.
Paper ModeSimulation mode where all trades are recorded but no real orders are placed. Default mode.

Risk Management

TermDefinition
DrawdownDecline from peak equity: (peak - current) / peak. A drop from $5,500 to $5,000 is 9.1% drawdown.
Heat System4-level progressive risk reduction: Normal (<10%, full sizing), Warning (10%, 50%), Critical (15%, 25%), Kill (20%, 0% — all trading halted).
Value at Risk (VaR)Maximum expected loss at a confidence level. VaR 95% = $150 means 19/20 days you lose less than $150. VaR gate blocks trades if portfolio VaR exceeds threshold.
Sharpe RatioRisk-adjusted return: mean(returns) / std(returns) × sqrt(252). Good: 1-2, Very good: 2-3, Excellent: 3+.
Sortino RatioLike Sharpe but only penalizes downside volatility. More relevant for trading.
Calmar RatioAnnualized return / max drawdown. Measures how well you handle worst-case scenarios.
Circuit BreakerAuto-disables an API after repeated failures. States: CLOSED (normal) → OPEN (failing, requests blocked) → HALF_OPEN (testing recovery).
Kill SwitchEmergency halt for all trading. Persistent across restarts. Accessible from dashboard, CLI, Telegram (/kill), Discord (!kill), and Slack.

Market Concepts

TermDefinition
Market RegimeAuto-classified market conditions: Normal (1.0x sizing), Trending (0.8x), Mean-Reverting (1.1x), High Volatility (0.6x), Low Activity (0.7x).
Conviction SignalSignal from 2+ quality-scored whale wallets taking the same position. Quality-weighted: S-tier whales count more than C-tier.
CLOBCentral Limit Order Book; Polymarket's orderbook for matching trades.
Gamma APIPolymarket's public REST API for market metadata and discovery.
WALWrite-Ahead Logging; SQLite journal mode enabling concurrent reads during writes.
